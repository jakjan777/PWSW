LAB-10  CWICZENIE 5.1
Diagram DFD i analiza STRIDE dla aplikacji audytowej SecurityScanner

================================================================================
DIAGRAM PRZEPLYWU DANYCH (DFD)
================================================================================

    [ Uzytkownik ]
          |
          v  (Granica 1: dane wejsciowe moga byc zlosliwe)
    [ WinUI 3 Desktop App ] -----> [ REST API (ASP.NET Core) ]
          |                              |
          v                              v  (Granica 2: komunikacja sieciowa)
    [ SQLite ]                    [ Foundry Local ]     [ Azure SQL DB ]
    (lokalna)                     (AI model)            (chmura)

Przeplyw danych:
  Uzytkownik -> WinUI 3: parametry skanu, raporty, konfiguracja
  WinUI 3 -> REST API: zadania audytu, zapytania o wyniki
  REST API -> Azure SQL DB: trwale dane audytu
  WinUI 3 -> SQLite: cache lokalny, historia skanow
  WinUI 3 -> Foundry Local: analiza AI modelu bezpieczenstwa

Powiazanie z modul 1 (SecurityScanner):
  SubsystemMonitor.CheckAsync symuluje health-check podsystemow
  Task.WhenAll / Task.WhenAny / SemaphoreSlim = rownolegle skanowanie
  FullScanRunner.RunFullScanAsync = pelny skan z timeoutem i postepem

================================================================================
GRANICE ZAUFANIA
================================================================================

G1  Uzytkownik <-> WinUI 3 Desktop App
    Ryzyko: zlosliwe dane wejsciowe, spoofing, XSS w polach formularza

G2  WinUI 3 <-> REST API
    Ryzyko: podszywanie sie, przechwycenie tokenu, MITM

G3  REST API <-> Azure SQL DB
    Ryzyko: SQL injection, wyciek danych, eskalacja uprawnien

G4  WinUI 3 <-> Foundry Local
    Ryzyko: prompt injection, wyciek danych treningowych, OOM

G5  WinUI 3 <-> SQLite (lokalna baza)
    Ryzyko: manipulacja pliku bazy na dysku, brak szyfrowania at-rest

TABELA ZAGROZEN STRIDE

#  STRIDE  Komponent              Zagrozenie                         Mitygacja                              Ryzyko

7   D       REST API               Przeciazenie REST API                Rate limiting, WAF                    Wysokie
8   D       Foundry Local          OOM przez duze zapytanie AI          Timeout, limit tokenow                Srednie

================================================================================
MAPA KOMPONENTOW DO KATEGORII STRIDE
================================================================================

Spoofing (S):          REST API (G2)
Tampering (T):         transmisja sieciowa (G2), SQLite na dysku (G5)
Repudiation (R):       brak audit log (G1, G2)
Information Disclosure: SQLite (G5), Foundry Local (G4)
Denial of Service:     REST API (G2), Foundry Local (G4)
Elevation of Privilege: WinUI MSIX (G1), SQL injection (G3)

================================================================================
PRIORYTETY MITYGACJI (skrot)
================================================================================

Natychmiast (Wysokie):  OAuth/JWT, HTTPS/TLS, rate limiting, parameterized SQL, AppContainer
Przed release (Srednie): DPAPI, audit log, szyfrowanie SQLite, timeout AI
Monitorowanie (Niskie):  filtrowanie output modelu AI
